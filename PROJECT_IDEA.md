# PROJECT_IDEA: Stolik

## 1. Проблема

Варшавские рестораны теряют 15-25% потенциальных гостей из-за хаотичного управления бронированиями:
- Бронь по телефону → хостес записывает в тетрадку → теряет, путает, забывает
- Существующие решения (TheFork, Booksy) берут комиссию 2-5€ за каждую бронь
- Нет единой системы: бронь на сайте не синхронизирована с телефонной книгой
- Маленькие рестораны (10-30 столиков) не могут позволить себе enterprise-решения (ResDiary, OpenTable — от $200/мес)

Гости:
- Звонят и не дозваниваются (особенно в пиковые часы)
- Не видят реальную доступность столиков
- Нет подтверждения брони — приходят и оказывается, что столик занят

## 2. Решение

**Stolik** — двусторонняя SaaS-платформа для бронирования столиков в ресторанах Варшавы.

Пошаговый процесс:
1. Ресторан регистрируется → проходит онбординг (инфо, часы, фото, столики/слоты) → публикует профиль
2. Гость открывает мобильное приложение → ищет ресторан (список, карта, фильтры) → выбирает дату/время/гостей → бронирует
3. Ресторан получает уведомление в реальном времени → подтверждает/отклоняет
4. Гость получает SMS-подтверждение + push-уведомление
5. За час до визита — напоминание
6. После визита — ресторан отмечает статус (completed/no_show)

Ключевое отличие: **подписка без комиссии за бронь**. Ресторан платит фиксированную месячную цену.

## 3. Почему сейчас

- Варшава — 4000+ ресторанов, рынок HoReCa растёт на 8% в год
- TheFork ушёл из Польши, ниша открыта
- Польские рестораны привыкли к подписочным моделям (Poster POS, Dotykačka)
- Expo + React Native позволяют выпустить кроссплатформу одним разработчиком

## 4. Целевая аудитория

**Основная — рестораны Варшавы:**
- Маленькие и средние (10-50 столиков)
- Задача: автоматизировать бронирования, снизить no-show, видеть аналитику
- Текущие инструменты: тетрадка, Excel, звонки

**Вторичная — гости:**
- 25-45 лет, живут в Варшаве или туристы
- Задача: быстро найти и забронировать столик
- Текущие инструменты: звонок, Google Maps, Instagram DM

```
              │  Socket.io          │
              └──────────┬──────────┘
                         │

              │  PostgreSQL         │

              └─────────────────────┘
                         │
         ┌───────────────┼───────────────┐
         │               │               │
    Cloudflare R2   Twilio SMS    Stripe Billing
    (фото)          (уведомл.)   (подписки)
```

**Стек:**
- **Mobile**: React Native + Expo (кроссплатформа одним разработчиком, EAS Build → TestFlight/Play Store)
- **Web Dashboard**: Next.js 16 (SSR, App Router, деплой на Vercel)
- **Backend**: Node.js + Express + Prisma (типизированный ORM, миграции)
- **БД**: PostgreSQL на Railway (managed, auto-backup)
- **Файлы**: Cloudflare R2 (S3-совместимый, дешевле AWS)
- **SMS**: Twilio (надёжность, польские номера)
- **Платежи**: Stripe (подписки, webhook, customer portal)
- **Real-time**: Socket.io (уведомления для дашборда)

## 8. План запуска

**MVP (сейчас → 2 недели):**
- Цель: 2-3 тестовых ресторана в Варшаве
- Метрики: ресторан принимает бронь через дашборд, гость бронирует через приложение
- Deliverable: TestFlight билд + работающий дашборд

**v1.0 (MVP + 1 месяц):**
- Stripe подписки активны
- SMS-уведомления
- 10+ ресторанов
- Виджет встроен на сайты ресторанов

**v2.0 (v1 + 2 месяца):**
- POS интеграция (Poster)
- Google Places отзывы
- Waitlist / очередь ожидания
- Android Play Store
